# Syncing this fork with upstream

`honeycombio/genai-prices` is a fork of [`pydantic/genai-prices`](https://github.com/pydantic/genai-prices).
Our **entire delta is the Go package** (`packages/go/`) plus a few small hooks that
keep its embedded data current:

- `packages/go/*` — net-new, never conflicts.
- `prices/src/prices/package_data.py` (`package_go_data`) — copies `prices/data.json`
  into `packages/go/data.json` during `make package-data`.
- `.github/workflows/ci.yml` (`test-go` job), `README.md`, `.prettierignore` — small additions.

Everything else — all price YAMLs, `prices/data.json`, the Python/JS packages — comes
straight from upstream and we never hand-edit it.

## Strategy: rebase our change onto the latest upstream

We keep the Go layer as a **clean patch replayed on top of `upstream/main`**, so history
stays linear. This means a force-push to `origin/main`; do it on a branch first and only
fast-forward `main` at the end.

### One-time remote setup

```bash
git remote add upstream git@github.com:pydantic/genai-prices.git   # if not already present
```

### Sync runbook

```bash
# 0. start from a clean tree — commit or stash local work
git stash

# 1. rebase the Go layer onto the latest upstream/main
git fetch upstream
git switch -c yingrong/sync-upstream main
git rebase upstream/main
#    Expected conflicts ONLY in: .github/workflows/ci.yml, README.md,
#    prices/src/prices/package_data.py — keep BOTH sides (our additions + upstream).
#    Price YAMLs and prices/data.json apply clean. Resolve -> git add -> git rebase --continue.

# 2. rebuild: YAML -> data.json -> python/ts/go embedded copies
make build
#    If packages/go/data.json changed, fold it into the Go commit to keep one clean patch:
#    git add packages/go/data.json && git commit --amend --no-edit

# 3. FORMAT-DRIFT GATE  (the important step)
git diff upstream/main -- prices/data.schema.json
#    - no diff -> the data format is unchanged; the Go structs are still valid. Continue.
#    - diff    -> the format changed. The diff names exactly which fields/types moved.
#                 Update packages/go/types.go (+ match.go / extract.go) to match,
#                 update knownKeys in packages/go/schema_drift_test.go, then re-run step 2.

# 4. verify
cd packages/go && go vet ./... && go test ./...   # schema-drift + golden guards run here
cd ../.. && make test                              # python suite

# 5. land it: open a PR (CI incl. test-go gates), then fast-forward main
git switch main
git rebase yingrong/sync-upstream                  # main == upstream/main + Go layer on top
git push --force-with-lease origin main
#    Cut a fresh release tag here if shipping a new Go module version.
```

## Why the format-drift gate matters

`packages/go/data.json` is a generated copy of `prices/data.json`, embedded via
`//go:embed`. The Go structs in `packages/go/*.go` that decode it are **hand-written**,
and `json.Unmarshal` is lenient: an upstream-**added** field is silently ignored, and a
**renamed** field decodes to a zero value. Either can make the Go package return wrong
prices with no error.

Two safety nets catch this:

1. **`prices/data.schema.json`** — regenerated by `make build-prices` from the Pydantic
   model. Diffing it against upstream pinpoints any format change (step 3 above).
2. **`packages/go/schema_drift_test.go`** — runs in the `test-go` CI job:
   - `TestSchemaDrift` compares every key in `data.json` against the keys the Go structs
     model (`knownKeys`). A new, renamed, or removed field fails the build loudly.
   - `TestGoldenPricesNonZero` asserts a couple of stable models still price to non-zero,
     consistent values — catching a renamed price field that decodes to a silent zero.

Type-level changes (a field changing JSON shape) already fail loudly: `init()` in
`data.go` panics, and the custom `UnmarshalJSON` in `match.go` rejects unknown match
operators. The gate + tests cover the _silent_ additive/rename cases those don't.

## Automation (future)

A scheduled GitHub Action can run steps 1–4 weekly, fail loudly if
`prices/data.schema.json` changed (forcing human review of the Go structs), and open a PR
for review. Defer until the manual runbook has run cleanly a couple of times.
